# AGENTS.md — Copiloto Pyme Chile

## Estado del repo

- **Fase actual: 3D.**

## Idioma

- Toda la documentacion, interfaces y copys del producto van en **espanol**.
- Los nombres de entidades y campos en base de datos estan definidos en ingles en `docs/data-model.md`; mantener esa convencion.

## Dominio (no negociable)

- **Pais:** Chile unicamente.
- **Usuario:** microempresas y emprendedores individuales.
- **Figuras legales soportadas:** Empresario Individual, EIRL, SpA (no SAS ni sociedades anonimas clasicas en MVP).
- **Regimen tributario:** PROPYME General con Contabilidad Simplificada.
- **No integrar con SII ni enviar formularios reales en el MVP.**

## Arquitectura clave

- Stack previsto: Next.js + TypeScript, Tailwind, Supabase (auth + PostgreSQL + pgvector), Zod.
- Ver `docs/architecture.md` para el stack completo y `docs/data-model.md` para el schema.

## Direccion visual

- `DESIGN.md` es la referencia visual oficial del proyecto.
- La landing pública en `/` está permitida; debe ser sobria y no exageradamente comercial.
- Las pantallas privadas bajo `/app/*` deben sentirse como una herramienta SaaS operativa, cálida, clara y confiable para microempresas chilenas.
- Dentro de `/app/*`: no usar heroes de marketing, gradientes decorativos ni UI ornamental.
- Al hacer cambios visuales, no cambiar contratos de datos, endpoints, schemas ni lógica de confirmación de acciones.

## Patron de accion segura (obligatorio)

La IA **nunca** escribe directamente en la base de datos. Flujo forzoso:

1. Usuario da instruccion natural.
2. Router clasifica intencion.
3. Agente genera propuesta estructurada.
4. Backend valida con Zod.
5. Usuario confirma.
6. Backend ejecuta.
7. Se registra auditoria en `agent_actions`.

## Plan de implementacion (secuencial)

No saltar fases. Ver `docs/implementation-plan.md`.

1. **Fase 1:** Prototipo web navegable (frontend estático/demo).
2. **Fase 2:** Backend local con schema real y endpoints.
3. **Fase 3:** Capa de IA controlada (router, schemas de salida, confirmacion humana).
4. **Fase 4:** Documentos y lectura basica de XML tributario.
5. **Fase 5:** Demo completa con flujo narrativo.

## Convenciones

- RAG con conocimiento curado, no volcado de leyes. Cada fragmento requiere fuente, fecha de vigencia y advertencia de riesgo si aplica.
- Toda recomendacion de IA debe incluir supuestos y fuentes.
- Mantener la trazabilidad: cada accion propuesta por IA se registra en `agent_actions` con `intent`, `confidence`, `model_used` y `sources_used`.
